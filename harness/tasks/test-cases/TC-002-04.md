## 前置条件
- PG16 + pgvector 起于 NAS；业务表含 `family_member_id` 且启用 RLS
- 会话以 `SET LOCAL` 注入 member claim（per-txn）

## 步骤
1. 成员 A 事务内写入若干行（claim=A）
2. 成员 B 事务内（claim=B）`SELECT *` 同表
3. 成员 B 尝试按已知 id 直查 A 的行
4. 无 claim/空 claim 事务读取

## 期望结果（对应 「PostgreSQL / RLS 策略生效：跨成员读取返回 0 行」）
- B 读取 A 的数据返回 **0 行**（API 与 DB 两层）
- 按 id 直查越权 → 0 行
- 无 claim 读取 → 0 行（默认拒绝）
