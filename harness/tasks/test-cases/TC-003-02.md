## 前置条件

- Keycloak ai-family realm 与 code+PKCE client 已配置，至少有 admin、adult、kid 三种角色和两个不同成员。
- GoalAgent 已关闭 X-Telegram-Chat-Id header 鉴权；MCP 网关和 goal-mcp 可接受 OIDC access token。

## 步骤

1. 分别为原 best_pal、go_getter 及 kid 身份走 code+PKCE 登录，取得合法 access token。
2. 校验 token 的签名、issuer、aud、exp、sub、成员 claim 与角色映射。
3. 分别调用读、成人写、管理写及 kid 自有打卡工具；再用 kid 调用成人/管理写工具。
4. 用缺失、过期、篡改 token 以及旧 X-Telegram-Chat-Id header 调用同一组工具。
5. 用成员 A 的合法 token 请求成员 B 的资源或工具参数。

## 期望结果

- best_pal 映射 admin，go_getter 映射 adult 或 kid；每个请求的成员身份来自验证后的 JWT。
- 工具在自身入口完成 JWT、角色和 tenant scope 校验，不依赖旧 header。
- kid 只能操作自己的 Draft-First 打卡；成人/管理类写入及跨成员访问均被拒。
- 缺失、过期、篡改 token 和旧 header 不能取得权限。
