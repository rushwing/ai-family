## 前置条件

- goal-mcp 注册 GoalAgent 的 36 个 tool；每个 manifest 含 risk、角色、tenant scope、两段式要求与测试引用。
- MCP 网关已启用 JWT、白名单、confirm token 来源校验和审计；旧 REST 与 OpenClaw 入口已禁用。

## 步骤

1. 枚举已注册 tool，并检查每个预期 tool 存在且每个写类 tool 的 manifest 字段完整。
2. 删除一个写类 tool 的 auth metadata 或测试引用后尝试注册。
3. 以成人身份调用 prepare→用户点击 confirm→execute 的写工具；分别尝试缺 confirm、伪造 confirm、Agent 自行生成 confirm。
4. 对同一工具尝试旧 REST/OpenClaw 直连、绕开网关的调用和跨成员参数。

## 期望结果

- 36 个 tool 的语义入口均经 goal-mcp/MCP 网关；写类 tool 有完整且可机检的 manifest。
- 缺 auth metadata 或测试引用的 tool 被拒绝注册。
- 只有来自 ChatUI 用户点击、与成员和来源绑定的 confirm token 能完成写入。
- 旧直连、绕过网关、伪造确认和跨成员调用均被拒，并留下审计记录。
