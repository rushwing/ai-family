## 前置条件

- ChatUI 的登录、对话、目标看板三屏与原型约定一致；Langfuse 测试项目可查询 trace。
- 已准备 adult 操作和 kid 结构化操作的测试账号。

## 步骤

1. 在浏览器中登录 adult，依次访问对话和目标看板，发起一个需要确认的目标写操作并由用户点击确认。
2. 从请求中记录 trace_id，查询 Langfuse 中 ChatUI、网关、agent-core 和工具调用的 span/metadata。
3. 登录 kid，验证只显示结构化入口和引导态；检查 UI 没有 M2 Planner/Plan Mode 的表述或入口。
4. 取消确认并重复请求，检查没有产生业务写入。

## 期望结果

- 三屏可用，确认只能由用户点击触发；取消或缺失确认不写入。
- 同一 trace_id 可贯通 ChatUI 请求、MCP 网关、GoalAgent 和工具调用，并可查 token、成本、延迟。
- kid UI 与结构化只路径一致，不呈现自由对话或创建/批准入口。
- UI 将确认描述为 GoalAgent 内部确认门，不暗示 M2 Planner 已接入。
