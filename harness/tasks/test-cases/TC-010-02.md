## 前置条件
- cloudflared 在 Pi5 出站长连接已建（无入站端口）
- CF Access policy 配置为家庭成员邮箱白名单，会话 24h

## 步骤
1. 外网浏览器访问公网域名（ChatUI 入口）
2. 未登录状态直接请求受保护路径
3. 用白名单邮箱完成 CF Access 登录后再访问
4. 对 Pi5 公网 IP 做端口扫描（443/80/22 等）
5. 用非白名单邮箱尝试登录

## 期望结果（对应 「CF Tunnel + Access / 域名可达，未登录被 Access 拦截，零入站端口」）
- 域名可达；未登录被 Access 拦截（跳转登录）
- 白名单登录后可达；ChatUI 侧二次校验 Access JWT 通过
- 端口扫描：**无任何入站开放端口**（流量仅经 cloudflared 出站隧道）
- 非白名单邮箱被拒

## 实现位置
- `infra/tests/cf_access_check.sh`（外网 runner + `nmap`/`curl`）
- CF 配置真值：`infra/cloudflare/`（access policy / tunnel）
