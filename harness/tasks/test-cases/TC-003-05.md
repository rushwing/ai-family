## 前置条件

- MinIO 测试 bucket、PG 归档版本表、notify.out broker/topic 与测试消费者已就绪。
- 创建报告归档和提醒发送各一笔可追踪的 GoalAgent 操作。

## 步骤

1. 归档一份周报，读取对象、版本表和同一报告的后续版本。
2. 模拟 GitHub 与 Telegram 直连客户端，观察归档和提醒操作产生的网络/adapter 调用。
3. 触发一条提醒，检查 notify.out 消息的成员、幂等键和消费者投递结果；重复投递同一消息。

## 期望结果

- 报告内容写入 MinIO，PG 版本表保存可追溯元数据与版本关系。
- M1 操作不调用 GitHub 归档或 Telegram 直连 client。
- 通知只经 notify.out 离开 agent-core；重复消息不会造成重复投递。
