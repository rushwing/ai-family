## 现象
adr_gate v2 只抓 `裁决: pending/待/未` 与「待裁决且无 human-001 裁决」两种形式；但标准更强：adr-standard §4.1 规则 2 要求 Review Notes「每条意见都带 → human-001 裁决」。于是一个 accepted ADR 只要写一条普通 bullet（如 `- [codex-x] 这个决策有问题`，不含待裁决/pending，且 informed_by 非空）即可通过 gate。

## 复现步骤 / 评审原文
- [codex-028][2026-06-16] severity P2（medium）。adr_gate.py line 63。建议直接逐 bullet 校验是否包含 human-001 裁决，defer 再额外校验 Revisit Trigger。

## 期望行为
每条 reviewer 意见都必须带 human-001 终裁；defer 须配 Revisit Trigger；记录/裁决类 bullet 豁免。

## 根因分析
v2 只针对 pending/待裁决两种"显式悬空"形式，未覆盖"普通未裁决意见"这一最常见情形。

## 修复方案（human-001 裁决：accept）
adr_gate v3 改为**逐 bullet 校验**：对 reviewer（`codex-*`/`gemini-*`）意见——① 含 pending/待裁决 → FAIL；② 无任何非-pending 的「裁决：」终裁 → FAIL；③ 裁决为 defer 但缺 Revisit Trigger → FAIL。`claude*`/`human-001` 等记录/裁决类 bullet 豁免（非"意见"）。残留：reviewer 自撤回/对齐类 bullet 也须补一条 human-001 裁决（已据此补 ADR-005 gemini-r2）。
