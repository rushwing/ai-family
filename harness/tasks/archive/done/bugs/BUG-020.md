## 现象
原型 Draft-First 弹层只展示任务与效果；ADR-010 要求 `prepare_xxx → confirm_xxx`，确认只能来自 ChatUI 用户显式点击并由网关校验来源。原型未表达"草稿已生成、确认令牌有效、确认后审计"的交互边界。

## 复现步骤 / 评审原文
- [codex-014][2026-06-15] severity medium。原型 line 294 弹层。关联 REQ-003 / ADR-010 / BUG-001 / BUG-011。

## 期望行为
高保真原型表达 confirm token 边界；kid 写动作显示"请求家长确认"而非同一确认按钮。

## 根因分析
弹层做了"草稿→确认"形，但未画 token 的 pending/有效期/来源/审计语义。

## 修复方案（human-001 裁决：accept）
卡片补：`pending` 状态、令牌有效期/来源提示、确认后审计提示；kid 写动作改为"请求家长确认"流（与 BUG-012/019 一致）。
