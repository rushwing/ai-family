## 现象
REQ-003 acceptance（line 13）写"家庭成员经 ChatUI 登录后可创建目标并完成一次打卡"，但同 REQ 后文（需求 7 / 非目标）已裁决 kid 不可自由对话、不可创建/批准目标写入。验收一句话会让 TC 设计误以为 kid 也要覆盖"创建目标"。

## 复现步骤 / 评审原文
- [codex-025][2026-06-15] severity low。关联 REQ-003 / BUG-012 / ADR-010。

## 期望行为
acceptance 明确角色边界，TC 不误覆盖 kid 创建目标。

## 根因分析
acceptance 早于 kid 结构化只路径裁决（BUG-012）。

## 修复方案（human-001 裁决：accept）
acceptance 改为"admin/adult 可创建目标；kid 可查看自有目标并完成 Draft-First 打卡"。

## 修复记录
- [claude-code-001][2026-06-15] 已按上述改 REQ-003 acceptance 一句话，角色边界与 BUG-012 一致。→ resolved。
