## 现象
08 §4 只说主图/wizard "共享 thread/checkpointer"，未定义 DB 业务表、LangGraph checkpoint、outbox 三者谁是恢复权威，以及 commit/checkpoint 失败交错时的恢复规则。上游 docs/langgraph-migration/02-comparison.md 实际选 GoalGroupWizard 表为 source of truth、checkpointer 只管控制流。

## 复现步骤 / 评审原文
- [codex-007][2026-06-15] severity high。关联 REQ-003 / BUG-001 / ADR-001 / ADR-003。

## 期望行为
明确状态权威与节点事务契约，崩溃恢复无歧义、不重复副作用。

## 根因分析
08 抽象层只讲"共享 checkpointer"，未落事务/恢复权威。

## 修复方案（human-001 裁决：accept）
08 §4/§5 补"状态权威矩阵"与节点事务契约：① 业务写入 + outbox + idempotency_key **同一 DB 事务**；② checkpoint 只记录控制进度（非业务真相）；③ 恢复时以 **DB 业务状态为权威** reconcile checkpoint/outbox；④ commit 成功但 checkpoint 失败 → 重放幂等去重。与 BUG-001 幂等设计同源。
