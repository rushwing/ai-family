## 现象
原型 line 63 链到 `#scope`，但 line 365 的 hash 解析只认 `#screen:role`，点击后回到默认 login/parent，破坏当前屏/角色定位。

## 复现步骤 / 评审原文
- [codex-017][2026-06-15] severity low。关联 REQ-003。

## 期望行为
评审定位链接不改变当前屏幕/角色。

## 根因分析
hash 路由未对非 screen hash 特判。

## 修复方案（human-001 裁决：accept）
对 `scope` 等非 `screen:role` 的 hash 特判：只 `scrollIntoView()` 不重置 state；或改为按钮滚动。
