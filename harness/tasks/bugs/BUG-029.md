## 现象
adr_gate v3 的 `RULING` 只校验"任意非-pending 的 `裁决：…`"，未要求 human-001；与脚本注释及 adr-standard §4.1 规则2（每条意见须带 **human-001** 裁决）矛盾。构造一条 `- [codex-x] … Claude 裁决：accept` 的 accepted ADR 能通过 gate。

## 复现步骤 / 评审原文
- [codex-029][2026-06-16] severity P2（medium）。adr_gate.py line 65。要求同一 reviewer bullet 上必须有 human-001，而非任意 `裁决`。

## 期望行为
reviewer 意见的终裁必须由 human-001 作出；`Claude 裁决`/裸 `裁决` 不算。

## 根因分析
`RULING` 正则未绑定 `human-001` 主语。

## 修复方案（human-001 裁决：accept）
`RULING = r"human-001\s*裁决[:：]\s*(?!pending|待定|未定|待|未)\S"`。校验：全量语料 reviewer bullet 本就用 `human-001 裁决：` 句式（扫描 0 例外），改后仍全过；负向自测 `Claude 裁决：accept` / 裸 `裁决：accept` 均判失败。

## 修复记录
- [claude-code-001][2026-06-16] adr_gate.py RULING 绑定 human-001；9 个 accepted ADR 通过 + 负向自测命中非人裁决。→ resolved。
