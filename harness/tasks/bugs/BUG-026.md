## 现象
`.github/workflows/ci.yml` 的 goal-agent lint 步骤同时有 `continue-on-error: true` 与 `ruff check agents/goal || true`。`|| true` 使该命令永远退出 0，GitHub Actions 连"失败但不阻塞"的可见信号都没有，变成静默绿；本 PR 又不跑 goal-agent 测试，等于隐藏了导入子树唯一的自动质量信号。

## 复现步骤 / 评审原文
- [codex-026][2026-06-15] severity P2（medium）。ci.yml line 52。

## 期望行为
迁移期 lint 可非阻塞，但失败必须可见。

## 根因分析
`|| true` 与 `continue-on-error` 叠用，吞掉退出码。

## 修复方案（human-001 裁决：accept）
去掉 `|| true`，保留 `continue-on-error: true`：ruff 失败时步骤标记 failed-but-non-blocking（可见、不阻塞合并），REQ-005 落地后转阻塞。安装 fallback `pipx install ruff || pip install ruff` 保留。

## 修复记录
- [claude-code-001][2026-06-15] 已改 ci.yml：ruff 步骤去 `|| true`，保留 continue-on-error；失败现为可见的非阻塞信号。→ resolved。
