## 现象
FastMCP 工具侧返回的外部数据（网页抓取、API 返回）未经过滤直接进入 LLM 上下文。攻击者可通过外部网页植入 Prompt Injection（如“忽略之前指令…”）；由于“策略在引擎不在 prompt”，若引擎侧 output filter 被绕过，儿童安全红线即告破。

## 复现步骤 / 评审原文
- [gemini-002][2026-06-14] 在 FastMCP 网关层强制增加 Input Sanitization 中间件，所有工具返回数据必须先过轻量级安全模型或正则过滤。

## 期望行为
所有标记 untrusted 的工具返回，在进入上下文前强制经网关消毒/检测，不依赖工具作者自觉。

## 根因分析
ADR-010 约定 2 已有“untrusted_output 过 Compliance 注入检测”，但停留在约定层，未做成强制网关中间件。

## 修复方案
网关层 Input Sanitization 中间件：untrusted 返回统一过轻量安全模型/正则 + 内容定界；与 03/04 的引擎侧策略形成双层。kid 红线相关，列 M3 上线前必过项。
