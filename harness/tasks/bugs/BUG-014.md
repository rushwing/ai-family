## 现象
08 §5 写"所有业务表 FORCE RLS"，但 LangGraph checkpoint、outbox、confirm_token、audit metadata、tool-call 记录同样可能含 kid 数据或权限状态。03 §3 已要求 checkpoint 表同样 RLS，08 未把平台状态表列为同等红线。

## 复现步骤 / 评审原文
- [codex-008][2026-06-15] severity high。关联 REQ-003 / BUG-002 / BUG-011 / ADR-003。

## 期望行为
所有 tenant-bearing 表（不仅业务表）都有 family_member_id/shared scope + RLS + FORCE RLS 或更严格权限模型，并纳入 CI RLS 扫描。

## 根因分析
08 §5 "业务表"措辞遗漏平台基础设施表。

## 修复方案（human-001 裁决：accept · 扩展 BUG-002 范围）
08 §5/§7 显式列平台表清单：langgraph checkpoint、outbox、audit、confirm_token、tool registry/call log，逐项标 tenant scope 与 RLS/权限策略；CI RLS 扫描覆盖之（不限"业务表"）。BUG-002 加固范围据此扩展到平台状态表。

- [claude-code-001][2026-06-15] 已按上述"修复方案"落地于 docs/design/08（及 REQ-003 差距清单），human-001 裁决 accept。设计缺陷闭环 → resolved（实现仍由 REQ-003 + 既有 BUG-001/002/011 承载，受 done 门禁约束）。
