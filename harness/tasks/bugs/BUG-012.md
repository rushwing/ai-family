## 现象
08 §3/§7 画的是 ChatUI/OIDC 直接进 GoalAgent，M1 最小基座列 PG+RLS+IdP+LiteLLM；但 01 §1、03 §1/§4、GLOSSARY 均要求 kid 输入/输出强制经过 Compliance（Compliance Agent 为 M3）。M1 若允许 kid 自由使用 GoalAgent，则缺最小合规门。

## 复现步骤 / 评审原文
- [codex-006][2026-06-15] severity high。只要 M1 允许 kid 使用 GoalAgent，就必须有 Compliance v0；否则 M1 明确禁 kid 访问。关联 REQ-003 / BUG-005 / ADR-010 / 03。

## 期望行为
kid 路径满足 P5 硬约束（kid I/O 受控），不引入未审 LLM 输出给 kid。

## 根因分析
08 把 kid 与 adult 同等接入 GoalAgent，但 M1 无 Compliance。

## 修复方案（human-001 裁决：accept · 取"结构化只路径"B 方案）
M1 kid = **结构化只路径**：RLS 下只读自有目标 + Draft-First 打卡 + praise_engine **离线模板**赞语（不走 LLM 自由生成）；**kid 不可自由对话 GoalAgent、不可创建/批准平台或成人目标写入**。如此无未审 LLM 输出给 kid → M1 无需 Compliance 亦满足红线。kid 自由对话 + 完整 Compliance v0 顺延 M3。08 §3/§7、REQ-003、原型 kid 态据此修订。BUG-005 维持 web_research 复活门（与本条独立）。

- [claude-code-001][2026-06-15] 已按上述"修复方案"落地于 docs/design/08（及 REQ-003 差距清单），human-001 裁决 accept。设计缺陷闭环 → resolved（实现仍由 REQ-003 + 既有 BUG-001/002/011 承载，受 done 门禁约束）。
