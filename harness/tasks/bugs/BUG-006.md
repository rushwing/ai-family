## 现象
harness/README 的评审回流规则说“选型异议写 ADR Review Notes”，但没有要求 proposed ADR 在 Review Notes 裁决后才能 accepted，也没有自动阻塞关联 REQ。ADR 异议可能记录了但不闭环。

## 复现步骤 / 评审原文
- [codex-009][2026-06-14] 增加 ADR review gate：informed_by、Review Notes 裁决、关联 BUG/REQ 全关闭后才能 accepted；REQ-001 验收应检查 ADR 状态一致性。

## 期望行为
ADR 状态流转受裁决闭环门禁；未闭环的异议阻塞 accepted 与其关联 REQ。

## 根因分析
ADR 状态机缺少“评审闭环”前置条件检查。

## 修复方案
ADR 标准补 gate；检查脚本/CI 校验 accepted ADR 无未裁决 Review Notes、关联 BUG/REQ 全闭；REQ-001 验收增 ADR 状态一致性检查。

## 修复记录
- [claude-code-001][2026-06-14] 三处落地：① adr-standard.md 新增 §4.1「评审闭环 Gate」（informed_by 完整 / Review Notes 全裁决 / 关联 BUG 全闭 / 决策变更同步正文，缺一阻塞关联 REQ）与 §4.2 一致性检查；② harness/README.md §5 补「ADR 评审闭环 gate」回流规则；③ REQ-001 验收明细新增第 4 条 ADR 状态一致性检查（存在未闭环 accepted ADR 则 REQ-001 不得 done）。gate 的自动化校验脚本/CI 留作 REQ-001 实施期落地。状态 → resolved，待 human-001 合并关闭。
- [claude-code-001][2026-06-14] 按 review（P1）细化 gate 并回填语料，消除「gate 立了但仓库自身不过 gate」的自相矛盾：① §4.1 规则 3 拆分——**决策一致性 BUG**（如 BUG-008）阻塞 ADR `accepted`，**实现加固 BUG**（带 `linked_req` 的 BUG-001/002/004）只阻塞下游 REQ 的 `done`，不回头打回已定决策；新增 `defer` 为合法裁决处置（须配 Revisit Trigger）；②§4.2 校验口径同步；③回填 7 个 accepted ADR（001/003/005/007/009/010/013）的 `informed_by`（从各自 Review Notes 来源 UID）；④ ADR-001 gemini-001「pending」重述为显式 `defer`+Revisit Trigger；⑤ README §5 与 REQ-001 验收第 4 条同步。复核中连带识别 **BUG-009**（06 状态图绕过鉴权网关，与 BUG-008 同类的决策一致性缺陷，按细化 gate 阻塞 ADR-010 accepted）并一并闭环（见 BUG-009 修复记录）。复核结论：现有全部 accepted ADR（001/003/005/007/009/010/013/014/015）满足细化后 gate——无「空 informed_by + 非空 Review Notes」、无悬空/pending 裁决、无未闭的决策一致性 BUG；剩余 open BUG（001/002/003/004/005/010/011）均为带 linked_req 的实现加固类，按 §4.1 仅阻塞下游 REQ 的 done，不阻塞任何 ADR。→ BUG-006 维持 resolved 且语料自洽。human-001 路线裁决：细化 gate + 回填（2026-06-14）。
