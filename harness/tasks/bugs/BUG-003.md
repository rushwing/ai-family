## 现象
ADR-002 把 LiteLLM 当预算硬顶与厂商 key 唯一持有者，但未定义 key 同步失败、预算计数延迟、网关宕机降级、“Agent 不直连”禁令如何验证。若只靠约定，后续 Agent 很容易绕开网关。

## 复现步骤 / 评审原文
- [codex-004][2026-06-14] 加 CI secret 扫描和 egress 限制，禁止 Agent 容器持有厂商 key；定义 LiteLLM 不可用时 fail-closed 还是降级；预算超限 TC 必须打真实网关。

## 期望行为
厂商 key 仅存在于网关；Agent 容器无任何厂商凭证且无法直连厂商；预算超限可靠阻断。

## 根因分析
强制点只在配置约定层，无技术门禁与出网限制。

## 修复方案
1. CI 扫描 Agent 镜像/配置不得含厂商 key；2. egress 白名单仅允许 Agent→LiteLLM，不放行厂商域名（挂靠 02 §2 I4）；3. 明确 LiteLLM 不可用 = fail-closed；4. 预算超限 TC 打真实网关验证。
