## 现象
08 §6 仅说 36 tools "语义不变"接入、write 补 risk/JWT、无 auth metadata/测试拒注册（组级），但无工具级清单，无法机器检查每个 tool 的 read/draft/write/high、是否有 auth decorator、是否 prepare/confirm、是否有回归测试。

## 复现步骤 / 评审原文
- [codex-011][2026-06-15] severity medium。关联 REQ-003 / REQ-004 / ADR-010。

## 期望行为
每个 tool 可被机器核验其风险级、角色、tenant scope、是否两段式、测试名；旧 REST/OpenClaw 直连入口在 M1 禁用或只走网关。

## 根因分析
08 处置粒度停在 6 组，未到 tool 级 manifest。

## 修复方案（human-001 裁决：accept · 与 REQ-004 衔接）
08 §6 增加 goal-mcp tool manifest（机器可读雏形）：逐 tool 列 risk / 角色 / tenant scope / 两段式(prepare,confirm) / 回归测试名；并声明 M1 旧 REST/OpenClaw 直连入口禁用、只走 MCP 网关。完整 manifest 由 REQ-004 注册门禁承载。
