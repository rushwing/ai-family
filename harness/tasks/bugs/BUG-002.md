## 现象
ADR-003 / docs/design/03 的 RLS 仅覆盖“正常应用查询”，未覆盖 SECURITY DEFINER 函数、迁移脚本、后台 worker、连接池复用、admin 运维脚本等绕过路径——这些才是 RLS 生产事故常见入口。

## 复现步骤 / 评审原文
- [codex-003][2026-06-14] 增加 RLS 安全规范：禁止默认 SECURITY DEFINER；迁移期用独立受限 role；worker 也必须设置 member/shared scope；CI 扫描新表 RLS/force RLS。

## 期望行为
任何数据访问路径（含后台 worker、迁移、运维）都不能绕过成员级行隔离。

## 根因分析
RLS 默认不作用于表属主 / BYPASSRLS / SECURITY DEFINER；连接池复用未重置会话变量会串号。

## 修复方案
1. 业务表 `FORCE ROW LEVEL SECURITY`；2. 迁移/运维用独立受限 role，禁 BYPASSRLS 进应用路径；3. worker 每任务 `SET LOCAL app.current_member`；4. 默认禁 SECURITY DEFINER，例外需评审；5. CI 扫描新表缺 RLS/FORCE RLS 即失败。
