## 背景
07/ADR-013 已识别 libs/ 变“上帝包”的风险，但反悔条件只有事后审查，没有前置边界和依赖规则（codex-008）。

## 需求描述
给 libs/ 分层（contracts / auth / state / agent-sdk），定义依赖方向（不得反向依赖业务、禁 libs/common），用 import-linter 做前置门禁。

## 非目标
不拆仓；不重构既有业务逻辑。

## 验收标准明细
- 定义并文档化 libs 分层与允许的依赖方向。
- import-linter 规则进 CI，违例即失败。
- 禁止新增 libs/common 之类无边界包。
