## 背景
07/ADR-013 的 path-filtered CI 没有依赖图来源；libs/** 改动触发“下游全量回归”需要知道哪些模块依赖哪个子包，否则漏测共享契约破坏（codex-007）。

## 需求描述
引入包管理/依赖图工具（如 uv / pnpm workspace + affected 检测，或 Nx/Turborepo/Pants），每个模块声明依赖；CI 按依赖图触发受影响模块的测试。

## 非目标
不引入重型构建系统（Bazel）除非必要；不改仓库边界。

## 验收标准明细
- 每个模块声明其对 libs 子包的依赖。
- 改 libs/auth 至少触发所有鉴权相关 Agent/MCP + e2e。
- 仅改单 Agent 时不触发无关模块测试。
