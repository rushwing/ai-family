## 背景
ADR-010 约定“每个 tool 首行校验 JWT”，但无共享 auth 库/强制中间件/注册门禁，会退化为作者自觉；36 个 goal tools 迁移漏一个即越权入口（codex-005）。

## 需求描述
把工具侧鉴权做成 FastMCP wrapper/decorator + 注册门禁：工具声明 auth metadata，网关注册时校验其存在与测试覆盖，缺失则拒绝注册。

## 非目标
不替换网关白名单/审计机制（ADR-010 既有）；不改 Draft-First 两段式。

## 验收标准明细
- 无 auth metadata 的 tool 注册被拒并报错。
- 无对应鉴权测试的 tool 注册被拒。
- write/high 风险工具统一经 decorator 注入 JWT 校验与 `app.current_member`。

## 设计引用
ADR-010 §约定 3 / ADR-003 RLS。
