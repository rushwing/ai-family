## 背景

goal-agent 现状：FastAPI + FastMCP（36 tools）+ MariaDB + Telegram + APScheduler，单用户角色模型
（X-Telegram-Chat-Id header 鉴权）。需迁移到平台基座并 LangGraph 化，成为第一个垂直切片。

## 需求描述（差距清单，详见 docs/design/06）

1. **数据迁移**：MariaDB → PostgreSQL 16（SQLAlchemy 驱动切换 + Alembic 基线重建 + 数据搬迁脚本），
   所有业务表加 `family_member_id` 并启用 RLS
2. **身份改造**：Header 鉴权 → OIDC JWT（IdP 签发），MCP tools 改为工具侧鉴权
3. **编排改造**：计划生成/跟踪重构为 LangGraph 长程图（goal → milestone → daily plan → check-in 循环），
   PG checkpointer 持久化，APScheduler 触发改为图的定时入口
4. **保留资产**：36 个 MCP tools 语义不变接入 MCP 网关；streak/XP/报表逻辑保留
5. **ChatUI v0**：NextJS 自研（以 Claude Design 设计稿为 scope 边界），登录 + 对话 + 目标看板三屏
6. **Dogfooding**：将本仓库 M0–M5 里程碑与 REQ 列表录入为第一个 Goal

## 非目标

- 不接 IntentRouter/Planner（M2）；Telegram 通道保留现状不增强；不做多语言

## 验收标准明细

1. 两个不同成员账号各建一个目标，互相在 API 与 DB 层面均不可见对方数据
2. kill 掉 Agent 进程后重启，进行中的月度计划任务从 checkpoint 恢复并继续
3. Langfuse 中可检索到从 ChatUI 请求到工具调用的完整 trace
4. GoalAgent 产出的首份周报包含 ai-family 项目真实进度
