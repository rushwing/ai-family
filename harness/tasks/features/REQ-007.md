## 背景
Gemini 评审指出的【最大单一缺口】：完全没有“LangGraph 陷入 ReAct 死循环（不断调用工具失败再重试）如何防止一晚刷爆 API 账单”的决策。家庭预算下这比系统宕机更致命。

## 需求描述
在 LiteLLM 网关层 + Agent 引擎层实现：基于 token/金额的硬性熔断（会话级/日级/成员级），ReAct 循环步数与工具重试上限，触发即 fail-closed 并告警。

## 非目标
不做精细的动态预算优化；先保底硬熔断。

## 验收标准明细
- 单会话超 token/金额阈值 → 后续 LLM 调用被拒。
- ReAct 步数 / 工具重试超上限 → 图终止并产出告警。
- 阈值可配置；超限事件写审计 + 通知。

## 设计引用
建议配套立 ADR-014（成本熔断决策）；强化 ADR-002 预算硬顶。
