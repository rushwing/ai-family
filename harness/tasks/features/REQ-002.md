## 背景

设计稿（REQ-001）定稿后，按 docs/design/02、03 落地平台基座最小集，为 GoalAgent 垂直切片（REQ-003）铺路。

## 需求描述（基座最小集清单）

| 组件 | 宿主 | 验收要点 |
|---|---|---|
| Tailscale mesh | 全节点（Pi5/Mac Mini/NAS/VPS） | 四节点互通，ACL 生效 |
| Cloudflare Tunnel + Access | Pi5 | 域名可达，未登录被 Access 拦截，零入站端口 |
| IdP（ADR-008 选型） | Mac Mini（618 前临时 Pi5） | 家庭成员账号 ×4，admin/adult/kid 三角色，OIDC 签发 |
| PostgreSQL 16 + pgvector | NAS | RLS 策略生效：跨成员读取返回 0 行 |
| Redis | Mac Mini（618 前临时 Pi5） | 会话缓存可用 |
| RabbitMQ | Mac Mini（618 前临时 Pi5） | 管理台可达（仅 Tailscale 内） |
| LiteLLM 网关 | Mac Mini（618 前临时 Pi5） | 路由 DeepSeek/Kimi/Claude 三后端，per-user 预算计数 |
| Langfuse | NAS | 一次 LLM 调用产生完整 trace（token/成本/延迟） |
| 备份链路 | NAS→VPS | PG 基础备份 + WAL 归档到异地，恢复演练通过一次 |

> **部署范围（三节点，对齐 docs/design/02 §1 与 ADR-009）**：Phase 1 目标形态为 Pi5（公网入口/轻量接入）+ Mac Mini M4（主运行时）+ NAS（数据节点）三节点。**618 前 Mac Mini 未到货**，运行时服务（IdP/Redis/RabbitMQ/LiteLLM/agent-core）临时落 Pi5，按 **Pi5+NAS 双节点降级验收**；Mac Mini 到货后将上述服务迁至主运行时节点并补三节点联调验收（不视为本 REQ 回归失败）。

## 非目标

- 不含任何 Agent 业务逻辑；不含 ChatUI（属 REQ-003 范畴）；不上 K3s（M5）
