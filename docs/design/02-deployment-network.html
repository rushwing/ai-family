3. Tailscale ACL（声明式网络策略 · 配置即文档）

// tags: infra=Mac Mini(runtime)/NAS/VPS, runtime=Mac Mini（运行时节点 · Mac Mini 同时持 infra+runtime 两 tag · ingress 唯一可达目标）,
//        ingress=Pi5（公网入口，最小可达面）, admin=家长管理设备, member=家人日常设备, kid=小孩设备
{
  "acls": [
    // 基础设施互通：仅白名单端口（PG 5432、Redis 6379、AMQP 5672、S3 9000、Neo4j 7474/7687、MCP 8xxx…）
    { "action": "accept", "src": ["tag:infra"], "dst": ["tag:infra:5432,6379,5672,9000,7474,7687,8000-8099"] },
    // ingress 最小化（BUG-008）：dst 限定 tag:runtime（仅 Mac Mini），非 tag:infra——
    //   故 ingress 触不到 NAS/VPS 的任何端口（含其 8000-8099），仅可达 runtime 的 agent-core/mcp；
    //   Pi5 被攻破时 blast radius 收敛到 runtime:8000-8099 单一目标，不能横向打穿 NAS 数据面或其余主机
    { "action": "accept", "src": ["tag:ingress"], "dst": ["tag:runtime:8000-8099"] },
    // 管理设备：可达全部管理面端口
    { "action": "accept", "src": ["tag:admin"], "dst": ["tag:infra:*", "tag:ingress:*"] },
    // 家人设备：仅应急直连 ChatUI/IdP（日常走公网域名）
    { "action": "accept", "src": ["tag:member"], "dst": ["tag:ingress:3000", "tag:infra:8443"] }
    // kid 设备：无任何 tailnet 访问规则（默认拒绝）—— 只能走 CF 入口，受完整身份链约束
  ],
  "ssh": [ { "action": "check", "src": ["tag:admin"], "dst": ["tag:infra", "tag:ingress"], "users": ["ops"] } ]
}
✅ 设计意图：ACL 文件进配置库，变更走 REQ —— 网络策略与代码同等对待（GitOps 纪律，为 M5 K3s NetworkPolicy 迁移预演）。
ingress→runtime 路由与 Subnet Router 风险（BUG-008）：Pi5（公网入口）与 Mac Mini（runtime）各自运行 tailscaled 作为独立 tailnet 节点，chatui→agent-core:8000 这一跳走上面的端口白名单规则；不使用 Tailscale Subnet Router——即不存在「Pi5 代理整个 192.168.110.0/24 家庭网段」的路径。因此即便面向公网的 Pi5 被攻破，攻击者经 tailnet 也只能触达 tag:ingress → tag:runtime:8000-8099（即仅 Mac Mini 的 agent-core/mcp 端口）——dst 限定 tag:runtime 而非 tag:infra，故 NAS/VPS 的任何端口（含其 8000-8099）对 ingress 均不可达，更无法直达 NAS 数据面（PG/Neo4j/MinIO）或内网其余主机，blast radius 由 ACL 收敛到单一 runtime 节点。M5 迁 K3s 后该跳由 Ingress Controller + NetworkPolicy 承接，安全语义不变（对应部署图 §1 的 ① ② 连线）。

4. 威胁面分析（家庭场景的现实排序）

威胁缓解措施残余风险
公网扫描 / 漏洞利用零入站端口（I1）；CF WAF；服务监听 tailnet/localhost低
ChatUI 认证绕过CF Access 前置 + ChatUI 二次验签（边界 2 双保险）；Keycloak PKCE低
提示注入（检索文档/网页/邮件内容夹带指令）untrusted 内容定界 + Compliance 输入扫描；openclaw-*/第三方工具输出强制标记 untrusted（ADR-010）；写动作 Draft-First 兜底——即使注入成功也无法直接执行中（LLM 固有）
越权访问他人数据（应用 bug）工具侧鉴权 + PG RLS 双层（边界 3）；跨成员读取 TC 为 P0 回归用例低
kid 绕过内容策略策略在引擎不在 prompt（P6）；kid 设备无 tailnet 规则；高风险工具对 kid 不可见而非拒绝中低
LLM API 数据外泄（出网内容）原则 P4 接受边界 + 出网白名单（I4）+ PII 上下文注入前最小化；高敏字段（持仓数量/成绩明细）不进 prompt中（接受）
SaaS 控制面（Tailscale/CF）被攻破数据面端到端 WireGuard；CF 仅见 ChatUI HTTP；管理面不在 CF 后低（接受）
NAS 固件/绿联云组件风险NAS 禁用厂商远程访问功能；平台数据全量可重建（备份在 VPS）中低
物理损坏/盗窃/勒索restic 加密异地备份 + 恢复演练（REQ-002 验收项）；备份仓库 append-only 模式防勒索软件追删低

5. 密钥与配置管理

  • Phase 1：secrets 用 sops + age 加密后进配置库（.env 不落明文 git）；compose 启动前解密注入。Phase 2（K3s）：sealed-secrets 接管，迁移路径已在 ADR-009 检查单。
  • 分发原则（I3）：每个 secret 只到达需要它的容器；轮换记录进 REQ；泄露应急 = 吊销 + 轮换 + 审计表回查使用记录。
  • 不自建 Vault：HashiCorp Vault 对家庭规模是运维负债，sops 即可达到"git 中无明文"的目标——此为有意识的降级，记录在 04 §3。
