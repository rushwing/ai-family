4. Compliance 设计与儿童角色策略矩阵

4.1 实现形态

  • 输入侧（所有角色）：规则层（敏感词/URL 黑名单/指令样式检测，毫秒级）→ 命中可疑再升级 LLM-judge（低价档，结构化输出 allow/deny/escalate + 理由）。两层设计控制延迟与成本。
  • 输出侧：kid 角色 100% 过 LLM-judge；adult 仅写动作与外发内容过。PII 脱敏在输出侧统一执行。
  • 策略来源：PG 策略表（策略引擎读取），版本化、家长在 Admin Console 可调——不写在 prompt 里（原则 P6）。
  • 拦截透明：对 kid 的拦截以引导性话术回复（不是冷冰冰的拒绝），同时事件入审计表，家长端可见摘要。
