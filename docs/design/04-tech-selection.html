领域选择落选项（败因一句话）关键 Trade-off（放弃了什么）反悔条件摘要
Agent 编排 ADR-001 ✅ LangGraph + ReAct + PG Checkpointer Google ADK（生态小、持久化弱）；自研 ReAct（造轮子违背 P1） 放弃 GCP 同源学习路径与自研透明度；接受框架版本风险（锁 minor） checkpointer 写入 p95>500ms；license 变化

知识图谱 ADR-005 ✅ Neo4j Community（受控本体 + GraphRAG-lite） PG+AGE（生态小众）；不建图谱（与显式需求冲突） 接受应用层隔离（弱于 RLS）——以"图谱不存高敏数据"对冲；接受第二引擎运维 需存敏感属性；NAS 内存压力
消息总线 ADR-006 🟡 RabbitMQ（quorum + DLQ） Kafka（任务语义要自己拼、Pi5 跑不动、对 <1msg/s 是教科书式过度设计）；PG 队列（手搓死信/监控） 放弃流处理学习机会（不强行塞入）；编排状态不走 MQ（LangGraph 职责） 出现真流式场景（行情 tick）单独评估
网络穿透 ADR-007 ✅ Tailscale mesh + CF Tunnel/Access FRP+VPS 自建（入口安全自担责，含 kid 的系统不试错）；纯 Tailscale（家人体验差） 接受两个 SaaS 控制面依赖与 CF 边缘 TLS 终止，换零公网端口 + 免维护入口 免费档政策变化
身份 IdP ADR-008 🟡 Keycloak 26（单 realm，3 角色） Authentik（多容器不更轻、企业特性弱）；Zitadel（生态薄，卡住没人问） 接受最重的资源占用与概念复杂度，换企业 IAM 全特性 + 最高学习价值 Pi5 内存持续>85% 且主因是 KC
部署形态 ADR-009 ✅ compose（M1–M4）→ K3s（M5）；数据层永不进集群 直接 K3s（双层调试拖慢切片）；只 compose（放弃学习目标） 接受一次有计划迁移返工；Phase 1 即遵守 12-factor/healthcheck/私有 registry 纪律降低迁移成本 M4 后 30 天未启动 M5 触发裁决
工具层 ADR-010 ✅ 一域一 FastMCP + 自研轻网关 + 工具侧鉴权 + Draft-First 单巨型 MCP（域耦合）；无网关直连（白名单/审计无处安放） 接受网关一跳与自研小组件；两段式写操作牺牲流畅度换人类确认硬保证 MCP 官方 registry 标准成熟；网关延迟占比>20%
对象存储 ADR-011 🟡 NAS 双形态：SMB（人用）+ MinIO（机用）+ restic→VPS 纯 SMB（无权限模型/审计）；MinIO 全接管（强迫家人改习惯） 接受学习资料双份存储与单向同步复杂度，换家人零迁移 + S3 标准接口 绿联系统变更；VPS 备份成本>R2/B2
可观测 ADR-012 🟡 Langfuse self-host + Prometheus/Grafana/Loki + PG 审计表 LangSmith SaaS（全家对话上云违背 P4）；纯 OTel 自拼（重造 Langfuse） 接受 Langfuse v3+ClickHouse 重量与三套系统维护，换数据不出家门 + eval 基建 NAS 内存不容纳（迁 MacMini）

事项选择理由 / 落选项

3. 有意不引入清单（防"为用而用"，每项给出回归条件）

技术（用户技能栈内）不引入理由什么时候回来
