05 — Rollout 计划与 ROI

里程碑时间线、每 Agent 的 shadow→draft→assisted 放量节奏（承自 reference 的 rollout playbook）、token 最大化利用策略、成本核算与 ROI。节奏单位是"周末档"（工作日有限投入 + 周末主力推进）。

1. 里程碑时间线

⚠️ 进度风险的结构性对策：里程碑录入 GoalAgent（M1 后），逾期由 GoalAgent 周报暴露给自己——用系统自己管理系统的开发（dogfooding 的真实价值）。M4 后 30 天未启动 M5 触发 ADR-009 的显式裁决。

5. ROI（双账本：家庭价值 + 学习价值）

5.1 家庭账本（reference 的 "hours saved" 框架）

  • 日程/目标管理：家长协调全家计划 ~2h/周 → GoalAgent 自动跟踪+报表，估省 1h/周
  • 学习辅导：作业答疑陪伴 ~5h/周 → StudyAgent 承接基础答疑（引导模式），估省 2–3h/周且质量更稳定
  • 旅行规划：每次出行收集攻略 ~8h → TravelAgent 收敛到 2–3h 确认型工作
  • 投研整理：盘后信息收集 ~3h/周 → StockAgent 备忘录化，估省 2h/周
  • 量化锚点（上线后用真实数据替换）：≈ 6–8 h/周 家庭时间回收，对照月成本 ¥95–200——按任何时薪折算都显著为正

5.2 学习账本（本项目的第二回报，与金钱无关但显式记账）

  • production 技能清单：LangGraph 长程编排 / MCP 工具治理 / OIDC+RLS 端到端租户隔离 / 混合检索与 eval / GraphRAG-lite / 零信任家庭网络 / K3s 迁移 / LLM 成本工程
  • 方法论样本：FDE 式垂直切片落地、ADR 决策留痕、shadow→draft→assisted 放量、harness 人机协作——每一项都有本项目的真实案例可复述（面试/分享可用）
  • 验收方式：每个里程碑结束写一篇复盘（存 KnowledgeAgent），M5 后输出一份"家庭版 → 企业版"差异分析作为学习闭环
